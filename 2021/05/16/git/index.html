Git的概念

  • Git 是一个免费的开源 分布式版本控制系统，旨在快速高效地处理从小型到大型项目的所有内容。（官方定义）

  • 简单点就是：Git 是目前世界上最先进的分布式版本控制系统（没有之一）。

  • Git 有什么特点？简单来说就是：高端大气上档次！

Linux 下的安装、命令行的使用

Git的安装

在使用 Git 前我们需要先安装 Git。Git 目前支持所有的主流平台上运行。

Git 各平台安装包下载地址为：http://git-scm.com/downloads

Ubuntu 通过： sudo apt-get install git 进行安装

在命令行输入 git 测试是否安装成功

Git的配置

Git 提供了一个叫做 git config 的工具，专门用来配置或读取相应的工作环境变量。

  • 配置用户信息

    • ```
      $ git config –global user.name “eddy”
      $ git config –global user.email c177xxxx@163.com

  • 配置差异分析工具

    • ```
      $ git config –global merge.tool vimdiff

Git基本操作

    - Workspace：工作区
    - Index / Stage：暂存区
    - Repository：仓库区（或本地仓库）
    - Remote：远程仓库

Git创建仓库

  • ​ Git 使用 git init 命令来初始化一个 Git 仓库，Git 的很多命令都需要在 Git 的仓库中运行，所以 git init 是使用 Git 的第一个命令。在执行完成 git init 命令后，Git 仓库会生成一个 .git 目录，该目录包含了资源的所有元数据，其他的项目目录保持不变。

  • ​ Git 克隆：使用 git clone拷贝一份远程仓库，也就是下载一个项目。

Git 基本操作

  • ​ 如果当前目录下有几个文件想要纳入版本控制，需要先用 git add 命令将文件存入缓存区

  • ​ 再通过git commit 将缓存区的内容存入本地仓库中

  • git status查看仓库当前的状态，显示有变更的文件。

  • git diff比较暂存区与工作区的差异

  • git rm 删除工作区的文件

  • git mv 移动或者重命名文件

git 查看日志

  • git log 查看历史提交记录

    • –oneline:历史记录的简洁的版本
    • –graph:查看历史中什么时候出现了分支、合并
    • –reverse:逆向显示所有日志

  • git blame 查看指定文件的历史修改记录

Git标签

git flow的应用

​ 为解决开发过程中缺乏清晰地流程与规范而提出的一种解决方案。

Git Flow 的常用分支

Git Flow 的使用

  • Master/Devlop 分支：所有在 Master 分支上的 Commit 应该打上 Tag，一般情况下 Master 不存在 Commit，Devlop 分支基于 Master 分支创建

  • Feature 分支：Feature 分支做完后，必须合并回 Develop 分支, 合并完分支后一般会删点这个 Feature 分支，毕竟保留下来意义也不大。

  • Release 分支：Release 分支基于 Develop 分支创建，打完 Release 分支之后，我们可以在这个 Release 分支上测试，修改 Bug 等。同时，其它开发人员可以基于 Develop 分支新建 Feature (记住：一旦打了 Release 分支之后不要从 Develop 分支上合并新的改动到 Release 分支)发布 Release 分支时，合并 Release 到 Master 和 Develop， 同时在 Master 分支上打个 Tag 记住 Release 版本号，然后可以删除 Release 分支了。

  • Hotfix 分支：hotfix 分支基于 Master 分支创建，开发完后需要合并回 Master 和 Develop 分支，同时在 Master 上打一个 tag。
