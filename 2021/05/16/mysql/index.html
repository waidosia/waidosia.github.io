数据库的范式

  1. 原子性：每列都是不可分割的基本数据项；是数据库的基本要求
  2. 唯一性：数据表中每条记录必须是唯一的
  3. 关联性：每个字段都与主键有直接关系，不存在传递依赖

字段约束

约束名称 关键字 描述
主键约束 PRIMARY KEY 字段唯一，不可为空
非空约束 NOT NULL 字段值不为空
唯一约束 UNIQUE 字段值唯一，可为NULL
外键约束 FOREIGN KEY 保证关数据的逻辑性

运算符

  1. 算术运算符

    加、减、乘、除、求模

  2. 比较运算符

    表达式 意义 例子
    IS NULL 为空 comm IS NULL
    IS NOT NULL 不为空 comm IS NOT NULL
    BETWEEN AND 范围 sal BETWEEN 2000 AND 3000
    LIKE 模糊查询 ename LIKE “A%”
    REGEXP 正则表达式 ename REGEXP “[a-zA-Z]{4}”

  3. 逻辑运算符

    表达式 意义 例子
    AND 与 age >18 AND sex = “男”
    OR 或 empno = 8000 OR deptno 20
    NOT 非 NOT deptno = 20
    XOR 异或 age >18 XOR sex = “男”

  4. 按位运算符

表达式 意义 例子
& 与 3&7
| 或 3|7
~ 取反 ~10
^ 异或 3^7
<< 左移 10<<1
>> 右移 10>>1

聚合函数

  • SUM函数:用于求和，只用于数字类型，字符结果为0，日期为毫秒数求和
  • MAX函数：用于获取非空值的最大值
  • MIN函数：用于获取非空值的最小值
  • AVG函数:统计数据的平均值
  • COUNT函数:COUNT(*)获得包含空值的记录数，COUNT(列名)用于获得包含非空值的记录数

    分组查询

    GROUP BY子句的作用是通过一定的规则将一盒数据集划分成若干个的小区域，然后针对每个小区域分别进行数据汇总处理

    逐级分组

    数据库支持多列分组条件，执行的时候逐级分组

要求：SELECT子句可以包含聚合函数，或者GROUP BY的分组列，其他内容均不可以出现在SELECT子句中

对分组结果集再次做汇总计算

GROUP_CONCAT函数

把分组查询中的某个字段拼接成一个字符串

Having 子句

类似于WHENE子句，在分组执行后执行

表连接

内连接是常见的一种表连接，用于查询多张关系表符合连接条件的记录;

外连接：除了符合条件的记录之外，结果集中还会保留不符合条件的记录
左连接与右连接：左外连接就是保留左表所有的记录,与右表做连接。如果右表有符合条件的记录就与左表连接。如果右表没有符合条件的记录,就用 NULL 与左表连接。右外连接也是如此。

UNION 关键字可以将多个查询语句的结果集进行合并 要求：字段个数相同，字段名称相同

子查询

子查询是一种查询中嵌套查询的语句
子查询可以写在三个地方：WHERE子句、FROM子句、SELECT子句。
WHERE 子查询最简单，最容易理解，但是效率是最低的

FROM 子查询只会执行一次，效率很高

SELECT 子查询:每输出一条记录就要执行一次，查询效率很低

单行子查询和多行子查询
WHERE子句中，可以使用IN,ALL,ANY,EXISTS关键字来处理多行表达式结果集的条件判断

    • CURTIMR():获得当前系统的时间，格式:hh:mm:ss

    • DATEDIFF(日期，日期)：计算日期的差值的天数

    • IF(表达式，值1，值2)
    • ```sql

      WHEN 表达式 THEN 值1
      WHEN 表达式 THEN 值2
      ……
      ELSE 值N
      END 事务的ACID属性
  • 原子性:所有操作只有2种状态，成功失败。不许停留在中间某个状态
  • 一致性:事务必须保证运行结果一致性
  • 隔离性:每个事务不受其他并发事务的影响
  • 持久性:一旦提交，便是永久的

| 序号 | 隔离级别 | 功能 |

| 1 | read uncommitted | 读取未提交数据(买票) |
| 2 | read committed | 读取已提交数据(银行) |
| 3 | repeatable read | 重复读取(电商) |
| 4 | serializable | 序列化
